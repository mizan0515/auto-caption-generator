# b31 — Whisper GPU 전환 스모크

Date: 2026-04-19
Script: `experiments/b31_gpu_smoke.py`

## 배경

`transcribe.py`는 `torch.cuda.is_available()` 분기로 GPU 경로를 타도록 작성되어
있으나 실제 파이프라인에서 `Whisper 로드 중: ... (device=cpu)`가 출력되었다.

원인:
- 파이프라인이 사용하는 Python312 환경에 **CPU 전용 `torch 2.11.0`** 이 설치되어 있었다.
- `torch.cuda.is_available() → False` → CPU 경로로 전사.

조치:
1. `pip uninstall torch torchaudio`
2. `pip install torch==2.11.0 torchaudio==2.11.0 --index-url https://download.pytorch.org/whl/cu126`
3. `transcribe.transcribe_audio` 마지막에 `torch.cuda.empty_cache()` 추가
   ([transcribe.py](../../transcribe.py) transcribe_audio 반환부).

- 전사 품질은 한국어 대화 정상 (환각 필터도 정상 동작).

## 적용한 변경

- 설치: Python312 `torch/torchaudio` → cu126 빌드.
- 코드: `transcribe.py` transcribe_audio 반환 직전 `empty_cache()` 호출 추가.
  긴 VOD(2시간+)에서 part 간 VRAM 누적 완화 목적. 8GB GPU 환경 안전 마진 확보.

## 후속 제안 (별도 실험 필요 — PROJECT-RULES 준수)

1. **batch_size 도입** — `transcribe_audio` 청크 루프를 batched generate로 변환.
   - 현 설정: `num_beams=5` + batch=1 → peak 1.8 GB.
   - batch=4 적용 시 peak 추정 4~5 GB (beam 수 × batch × feature tensor). 8 GB GPU에서 가능은 하지만 VOD 길이·audio 복잡도에 따라 OOM 위험. → 별도 `b32_whisper_batch.py` 로 측정 후 적용 여부 결정.
   - 기대 효과: 청크별 1초 → batch=4 기준 0.3~0.4초. 1시간 VOD 전사시간 2~3x 단축 여지.

2. **num_beams 재튜닝** — beam=5는 품질 보수 설정. turbo 모델 + 한국어에서 beam=3 또는 beam=1+temperature fallback 으로도 품질 유지 가능성. 별도 A/B 측정 필요.

3. **dashboard 재시작** — 현재 돌고 있는 `pipeline.dashboard` 프로세스는 이미 구 torch 참조를 import했을 가능성이 있다(실제로는 torch import 없음 확인). 새 전사 프로세스는 재설치된 torch를 로드하므로 추가 조치 불필요.
